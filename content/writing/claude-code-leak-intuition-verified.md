Claude Code 意外洩漏原始碼之後，看到社群普遍的反應是佩服。

老實說我不意外。

我一直有個直覺——CC 好用絕對不只是因為模型厲害。舉個例子，就像 Mac 和 Windows 的差別，不是硬體規格的問題，是整個系統的設計邏輯不一樣。洩漏的程式碼某種程度驗證了這件事：工具本身就有獨立於模型的競爭力，架構設計是精心打造過的，不是隨便包一層 CLI 就上線。

不過真正讓我更感興趣的是第二件事——dogfooding。

Anthropic 內部有個詞叫「antfooding」（因為員工暱稱是 ant），根據 CC 產品負責人的說法，70 到 80% 的技術員工每天都在用 CC 工作，內部回饋頻道大概每五分鐘就有一則新貼文。更關鍵的是，CC 會自動使用最新的研究模型快照，讓員工直接嵌在模型開發的回饋迴路裡面。這代表工具和模型之間不是各做各的，而是緊密地互相影響——模型變強了，工具跟著變好用；工具的回饋又反過來幫模型調整方向。這種垂直整合，不是其他公司拿同一個模型就能複製的。

這就是一個飛輪。用→發現問題→改→更好用→用更多→發現更多問題。一旦轉起來，外面的人很難追。

做字的人可能比較有感：想像一個字型設計師，每天都用自己設計的字排文件、寫信、做簡報。kerning 哪裡卡、哪個字重在小字不好讀，他一定比任何外部測試都更早發現。別人可以拿到他的字型檔，甚至可以反編譯去研究，但複製不了那個「每天用、每天改」的迴路。

其實我自己就是這樣。九宮格預覽 Pro 就是從我自己做字的需求長出來的——開源版用了一陣子，覺得不夠，就一路改到變成產品。最近在做的注音鍵盤 APP 也是同一個模式，手指粗按錯鍵是我自己的痛點，所以自己做了一個，每天用每天改。

複製成品很容易，複製這種迴路很難。我覺得這才是 CC 真正的護城河。

然後洩漏的程式碼裡還有一個細節讓我覺得蠻有趣的。CC 的創辦人 Boris Cherny 說他自去年 11 月起沒有手動編輯過一行程式碼，100% 由 CC 自己撰寫。社群有人因此質疑程式碼品質，但我不覺得這是問題——Boris 有近乎無限的 usage 資源和深厚的工程背景，他的工程知識和 AI 能力高度堆疊，本身就形成了一套品質把關機制。他有條件把控制權完全交出去。

老實說，如果我有同樣的條件，我大概也會走到那個位置。

這是[在光譜上的理性選擇](/writing-post.html?slug=vibe-coding-to-enlightened-autocracy)，不是偷懶。每個人在這條光譜上的位置，從來都不是固定的，而是要持續校準的。
