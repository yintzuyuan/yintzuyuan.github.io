說實話，一個月前剛開始重寫九宮格預覽時，我原本打算維持開源免費。

當時的想法很簡單：把 Python 版的功能，用新架構復刻一次就好。雖然 Python 用了兩年，但開發體驗對我來說一直是種折磨——不斷重啟 Glyphs 抓錯誤訊息、過於自由的寫法，以及難以直觀編寫的 UI，讓我的進步非常緩慢。我常想，也許不是 Python 做不到，而是這種開發模式限制了我的想像力。

但當我切換到 Objective-C，並開始與 AI 協作後，事情失控了。往好的方向。

不知道是因為原生語言的嚴謹結構，還是我終於摸索出駕馭 AI 的甜蜜點，開發過程突然變得異常順暢。那種感覺就像是打通了任督二脈：

「既然寫得這麼順，那以前不敢想的『拖曳鎖定』能不能做？」——可以。

「老用戶許願很久的『字組記憶』能不能加？」——做到了。

我越做越多，功能越疊越深。等到回過神來，它已經遠遠超越了原本的「復刻」，變成了一個兩年前無法想像的全新產品。

所以，NineBoxView Pro 誕生了。

這不僅僅是一個外掛的重寫，而是我這一個月來，在開發愉悅感中「溢出來」的成果。原本的 Python 版依然開源，它依然是個好工具。但如果你想體驗那些我「終於實現」的功能與手感，Pro 版在那裡等你。

## 造字工具宇宙的延伸

在重構九宮格的同時，另一塊拼圖也補上了。

四年前，我曾經在漢字造字系統實驗的 VLOG 裡分享過一個「漢字樹狀結構」的造字計畫。當時我希望能有一個工具，幫助我像剝洋蔥一樣，從筆畫、部件到整字，有邏輯地搜尋與管理漢字。但那時候卡在技術門檻，很多想法只能手動硬刻，變數難以控制，做得很辛苦。

這四年來計畫看似擱置，但我從來沒有忘記那個「讓設計師快速找到相關字」的執念。最近因為重構九宮格提高了 AI 協作產能，就一起把這個工具——漢字 IDS 部件查詢——處理起來了。

它解決了我們做字時常遇到的一個難題：「如何快速找到結構相似的字？」

這不是一個完美的最終產品，而是我的「造字工具宇宙」裡正在生長的一個有機體。它代表了我從純手工造字，轉向利用 AI 輔助開發工具的過程。九宮格處理的是漢字的「視覺空間」佈局，部件查詢處理的是「結構邏輯」拆解——兩個角度，剛好互補。

這個工具現在已經開源了，希望能幫助同樣在字海中探索的你。
